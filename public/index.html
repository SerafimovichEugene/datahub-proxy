HOW IT WORKS

EPAM has dozens of internal systems that produces and consumes data. Historically, most of integrations where built as peer-to-peer network which complicated the overal integration topology and decrease effectiveness of company resources. The Data Hub addresses this challenge, so the company can simplify design, development and operations (as well as reduce time-to-market) of new capabilities and its costs.

Our integration bus is intended for transferring data between applications, so developers can focus on data usage, but not worry about how to share or access it. This distributed messaging is based on Publisher/Subscriber pattern and built on Apache Kafka: messages are persisted in a topic and apps queue them asynchronously which gives every participant plenty of flexibility.

The multi-temporal data lake serves as an accumulator of internal EPAM data. It supports browsing of temporal data for analytical and reporting purposes. The system is designed as a PostgreSQL database (some data are stored in relational model, some in plain json format) with additional components for handling historical data. As of December-2017, the data lake stores more than 1Tb of data which is replicated from more than 15 EPAM systems.
